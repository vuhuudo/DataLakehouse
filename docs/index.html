📖 What is DataLakehouse?

DataLakehouse is a complete, production-ready data stack that runs entirely on Docker Compose. It implements the Medallion Architecture (Bronze → Silver → Gold) to manage data quality and complexity, integrates automated ETL pipelines with Mage.ai, serves analytics queries through ClickHouse, and provides interactive dashboards via Apache Superset.

Designed for local development, testing, and small-to-medium production deployments, DataLakehouse includes containerized databases, workflow orchestration, monitoring, authentication, and best-in-class data tools — all in one Docker Compose stack.

✨ Key Features

🏗️ Medallion Architecture

Bronze → Silver → Gold layers enforce data quality and lineage tracking. All data persists in RustFS object store as Parquet files.

🔄 Automated ETL

Mage.ai orchestrates data pipelines with visual workflow builders, triggers, and scheduling. Load from PostgreSQL, CSV, Excel, and more.

⚡ OLAP Analytics

ClickHouse columnar database serves fast analytics queries. Supports complex aggregations, window functions, and real-time data.

📊 Business Intelligence

Apache Superset provides interactive dashboards, SQL queries, and data exploration. No coding required for business users.

🔒 Identity & Security

Authentik provides centralized SSO, RBAC, and user management. Optional TLS reverse proxy with Nginx Proxy Manager.

📈 Monitoring & Logging

Grafana tracks pipeline health and performance. Deep health checks and diagnostic tools for troubleshooting.

🛠️ Technology Stack

Data Flow

PostgreSQL / Excel / CSV │ ▼ [Mage ETL] RustFS Bronze → Silver → Gold (Parquet, partitioned by date) │ ▼ [Serving Layer] ClickHouse analytics DB │ ├──▶ Superset (business dashboards) └──▶ Grafana (pipeline monitoring)

Components by Layer

🗂️ Ingest Layer

PostgreSQL Source database & staging RustFS Console Object store UI CSV / Excel Upload File-based ingestion

💾 Storage Layer

RustFS (S3-compatible) Parquet files in Bronze/Silver/Gold buckets Redis Stack Cache & queue

⚙️ Processing Layer

Mage.ai ETL orchestration & workflow builder CloudBeaver Web-based SQL IDE

📊 Warehouse Layer

ClickHouse Columnar OLAP engine for analytics

🎨 Analytics Layer

Apache Superset Interactive dashboards & BI Grafana Monitoring & alerting

🔐 Infrastructure Layer

Authentik Centralized SSO & RBAC Nginx Proxy Manager TLS reverse proxy (optional) Dockhand Docker management UI
ℹ️ Design Principle: The system enforces a strict Medallion architecture where all data must reside in the RustFS object store (the single source of truth) before any transformation. ClickHouse can be fully rebuilt from the lake at any time.

🚀 Quick Start

Prerequisites

  • Linux / macOS / WSL host
  • Docker Engine + Docker Compose plugin
  • uv (host-side Python runner)
docker --version docker compose version uv --version

Step 1: Clone & Install

git clone https://github.com/HoangThinh2024/DataLakehouse.git cd DataLakehouse uv sync --all-groups

Step 2: Run Guided Setup (Recommended)

bash scripts/setup.sh

The setup script will:

  • Prompt for configuration values (or accept defaults)
  • Create a complete .env file
  • Set up Docker network and start all services
  • Optionally load sample data and provision dashboards

Step 3: Verify Stack Health

bash scripts/stackctl.sh health # deep health checks bash scripts/stackctl.sh diagnose # analyze logs and conflicts

All services should report healthy within 2–3 minutes of startup.

Step 4 (Optional): Load Sample Data

uv run python scripts/run_etl_and_dashboard.py
💡 Tip: For non-interactive (CI) mode, add the --auto flag to skip prompts.

Day-2 Operations

All lifecycle management goes through scripts/stackctl.sh:

bash scripts/stackctl.sh up # start all services bash scripts/stackctl.sh down # stop all services bash scripts/stackctl.sh status # container status bash scripts/stackctl.sh health # deep health checks bash scripts/stackctl.sh logs all # stream all logs bash scripts/stackctl.sh reset # soft reset (keep volumes) bash scripts/stackctl.sh reset --hard # hard reset (delete volumes)
